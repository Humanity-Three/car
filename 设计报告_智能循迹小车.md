# 智能循迹小车 — 设计报告

## 一、系统概述

基于 **TI MSPM0G350x** 微控制器的智能循迹小车。通过 **8 路灰度传感器**检测地面黑线，**MPU6050 六轴陀螺仪**获取偏航角辅助导航，**双路正交编码器**实现闭环速度控制与里程计量，**OLED 显示屏**实时显示运行数据。支持四种路径模式，通过按键切换和启动。

---

## 三、软件架构

### 3.1 整体流程图

```
系统上电
  ↓
SYSCFG_DL_init() — 初始化系统时钟、GPIO、定时器、PWM
  ↓
外设初始化：OLED / MPU6050 / Grayscale_Sensor
  ↓
主循环（while(1)，~10ms/周期）
  ├── 读取灰度传感器（Grayscale_Sensor_Read_All）
  ├── 读取 MPU6050 角度（MPU6050_Get_Angle）
  ├── OLED 刷新显示
  ├── 按键扫描：
  │   ├── S2 短按 → 切换模式（mode = 0~3）
  │   ├── S2 长按（≥1.9s）→ Yaw 清零
  │   └── S1 按下 → 启动运行（激活计时器）
  └── 根据 mode 调用对应控制函数
        ├── mode=0 → Control_AB()
        ├── mode=1 → Control_ABCDA()
        ├── mode=2 → Control_ACBDA()
        └── mode=3 → Control_ACBDAx4()
```

## 四、核心控制算法

### 4.1 闭环速度控制 — 增量式 PID

采用增量式 PID 对左右轮分别进行速度闭环控制：

```
Pwm += Kp * (Bias - Last_bias) + Ki * Bias + Kd * (Bias - 2*Last_bias + Last2_bias)

参数：Kp = 0.9, Ki = 0, Kd = 0
```

经实际测试，仅比例控制即可满足循迹需求，积分和微分项置零。目标速度为 `Speed_Middle ± bias`，其中 `bias` 为转向差速值。

### 4.2 灰度循迹算法

8 路传感器（P1~P8）从车体左到右排列，加权求取偏差：

```
xunji() 返回值（bias）：
  P4≠0 → -10          P5≠0 → +10
  P3≠0 → -18          P6≠0 → +18
  P2≠0 → -22          P7≠0 → +22
  P1≠0 → -34          P8≠0 → +34
  全白 → 0

bias > 0 → 左轮加速、右轮减速 → 右转
bias < 0 → 左轮减速、右轮加速 → 左转
```

负值对应左转（P1~P4 检测到黑线），正值对应右转（P5~P8 检测到黑线）。内侧传感器优先响应，差速值从内向外递增。

### 4.3 白色区域导航 — 陀螺仪辅助

当小车行驶到白色区域（无黑线可循）时，依靠 MPU6050 的偏航角 Yaw 进行方向修正：

```
bias = Normalize_Yaw_Error(Yaw, target_yaw)

角度归一化处理：
  diff = current - target
  while diff > 180° → diff -= 360°
  while diff < -180° → diff += 360°
```

#### 多阶段白色直行策略（ACBDA 模式）

借鉴"斜线→摆正→直行"三段式设计：

| 阶段 | 行为 | 切换条件 |
|------|------|---------|
| **Phase 0** | 目标偏航 = 起始角 ± 37.5°，沿对角线行驶 | 编码器里程达到 `ACBDA_DIAGONAL_TICKS`（≈4100 脉冲，对应约 1.28m 斜边） |
| **Phase 1** | 目标偏航摆正到直行方向 | 偏航角误差 < ±5° |
| **Phase 2** | 直行等待黑线出现 | 灰度传感器检测到黑线 |

`n` 为奇数时（A→C 方向），目标角度为 `start_yaw - 37.5° → start_yaw`；
`n` 为偶数时（B→D 方向），目标角度为 `start_yaw + 180° + 37.5° → 180° - start_yaw`。

### 4.4 模式切换防抖

为防止传感器噪声导致误切换，设计了双重防抖机制：

**白→黑切换**：
- 仅当 `black_count > 0` 且连续 16 次采样都为非全白，才实际切到循迹模式
- 切到循迹后，还需连续 20 次全白才能重新允许进入黑线

**首次黑线过滤器**：
- 白色区域刚进入黑色弧线时，若 `white_phase < 2` 阶段检测到少量黑线，保持白色模式不变，防止误触发

## 五、四种运行模式

### 5.1 模式 0：Control_AB（A→B 直线）

| 项目 | 说明 |
|------|------|
| **路径** | 从 A 点沿直线行驶到 B 点黑线 |
| **控制方式** | 仅陀螺仪保偏直行（`bias = Yaw`，±2°死区） |
| **停止条件** | 任意灰度传感器检测到黑线 |
| **特点** | 最简单的直线行驶模式，用于验证陀螺仪和电机 PID |

### 5.2 模式 1：Control_ABCDA（矩形路径）

| 项目 | 说明 |
|------|------|
| **路径** | A→B（白）→ C（黑弧）→ D（白）→ A（黑弧）|
| **白色段** | 陀螺仪保偏直行 |
| **黑色弧线** | `xunji()` 循迹 |
| **停止条件** | D→A 弧线结束后，检测到白色区域 |
| **特点** | 典型矩形赛道，白-黑交替各两次 |

运行流程：

```
A点启动
  │
  ├── 白 A→B：陀螺仪保持 start_yaw 直行
  │     ↓
  ├── 黑 B→C：xunji() 循迹，black_arc_ticks 计数
  │     ↓
  ├── 白 C→D：陀螺仪保持 180°-start_yaw 直行
  │     ↓
  └── 黑 D→A：xunji() 循迹，black_arc_ticks≥20 且 n≥3 时准备停车
        ↓
  回到 A 点 → 停车
```

### 5.3 模式 2：Control_ACBDA（交叉路径）

| 项目 | 说明 |
|------|------|
| **路径** | A→C（斜线）→ B（弧线）→ D（斜线）→ A（弧线）|
| **白色段** | 三段式：对角线→摆正→直行 |
| **黑色弧线** | `xunji()` 循迹 |
| **里程控制** | 编码器累计里程控制对角线距离 |
| **停止条件** | n≥3 且进入白色区域 |

运行流程：

```
A点启动
  │
  ├── A→C（白）：Phase0 斜线 → Phase1 摆正 → Phase2 直行
  │     ↓
  ├── C→B（黑弧）：xunji() 循迹，n=2
  │     ↓
  ├── B→D（白）：Phase0 斜线 → Phase1 摆正 → Phase2 直行
  │     ↓
  └── D→A（黑弧）：xunji() 循迹，n=3
        ↓
  n=3 时进入白色 → 停车
```

### 5.4 模式 3：Control_ACBDAx4（交叉路径 ×4 圈）

与 ACBDA 逻辑相同，但连续执行 4 圈：

| 圈数 | n 范围 | 路径 |
|------|--------|------|
| 第 1 圈 | n=1,2 | A→C→B→D→A |
| 第 2 圈 | n=3,4 | A→C→B→D→A |
| 第 3 圈 | n=5,6 | A→C→B→D→A |
| 第 4 圈 | n=7,8 | A→C→B→D→A |
| **停车** | **n=9** | **第 4 次 D→A 弧线结束 → 停车** |

## 七、技术特点总结

1. **灰度加权寻迹**：8 路传感器加权平均定位黑线中心，非简单的阈值开关，转向平滑
2. **陀螺仪辅助导航**：白色区域无黑线可循时，依靠 MPU6050 偏航角保偏直行，解决丢失黑线的方向问题
3. **里程计辅助控制**：编码器累计里程用于对角线距离控制，不依赖纯时间延迟，适应性更强
4. **角度归一化**：`Normalize_Yaw_Error` 将角度误差归一化到 ±180°，解决 0°/360° 跳变问题
5. **多阶段白色策略**：斜线→摆正→直行的三段式设计，比直接直线行驶更平滑地过渡到目标方向
6. **多模式可切换**：4 种路径模式通过按键切换，长按 Yaw 清零，便于调试和验证
7. **双闭环控制**：编码器速度闭环（增量式 PID）+ 陀螺仪方向闭环（P 控制），实现稳定的速度与方向控制

## 八、调试与显示

OLED 实时显示信息：

```
第一行：运行模式（AB / ABCDA / ACBDA / ACBDAx4）
第二行：运行时间（time: xx.xx 秒）
第三行：偏航角（Yaw: xx.xx°）
第四行：8 路灰度值（G: 00001111）
右上角：运行指示星号闪烁
```

按键功能：
- **S2 短按**：循环切换模式（0→1→2→3→0）
- **S2 长按（≥1.9s）**：Yaw 角归零
- **S1 按下**：启动/重新开始运行
